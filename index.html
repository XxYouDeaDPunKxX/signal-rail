Signal Rail

Think. Throw. Unearth.

A text-first governance system for AI-assisted project work.

Signal Rail helps an AI enter the work with the right frame, sort project material with more care, and keep it recoverable without flattening everything into one documentary surface.

03

Unearth

The right line can resurface from the right layer later, without manual reconstruction and without pretending memory is enough.

The agent is not corrected after the fact. It is routed before acting.

Workstation

An optional offline workstation for direct canonical work.

What it does

It gives the operator a direct local workstation for reading canonicals, checking runtime state, staging writes, previewing output, and writing back to the matching .txt files when live folder access is available.

What it does not change

It does not replace the canonicals or change their authority. It gives the operator a clearer local way to maintain a live instance without working blind.

Failure mode

Context collapse.

Signal Rail exists to prevent current work, won decisions, unresolved material, technical topology, and continuity from collapsing into the same documentary surface.

  • live state mixed with decisions
  • unresolved material promoted too early
  • technical reality buried inside generic notes
  • continuity pretending to be project truth

Kernel

Protocol

How the system keeps the AI steady.

enter validate entry activate close frame read minimum classify write if clear
  • 00_runtime_entry.txt first, 06_ai_to_ai.txt after entry is valid
  • one way of working at a time
  • read enough before acting
  • stop if the frame, meaning, or destination is still unclear
